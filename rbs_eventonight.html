Espansione della piattaforma EvenToNight

  • 2 App Mobile
    • 2.1 Interfaccia Utente
      • 2.1.1 Home
        • 2.1.1.1 Il Sistema Deve Suggerire eventi agli utenti non registrati
        • 2.1.1.2 Il Sistema Deve Suggerire eventi personalizzati agli utenti registrati
      • 2.1.2 Esplora
        • 2.1.2.1 Il sistema deve permettere agli utente di cercare eventi e utenti
        • 2.1.2.2 Il Sistema in assenza di filtri e ricerche attive deve suggerire eventi agli utenti non registrati
        • 2.1.2.3 Il Sistema in assenza di filtri e ricerche attive deve suggerire eventi personalizzati agli utenti registrati
        • 2.1.2.4 Il Sistema deve permettere all'utente di filtrare gli eventi
      • 2.1.3 Pagina dell'evento
        • 2.1.3.1 Il Sistema Deve Mostrare i dettagli dell'evento
        • 2.1.3.2 Il Sistema Deve Permettere la visualizzazione delle recensioni (se presenti) nel caso in cui l'evento sia concluso
        • 2.1.3.3 Il Sistema Deve Permettere agli utenti registrati di acquistare i biglietti per l'evento
        • 2.1.3.4 Il Sistema Deve Permettere agli utenti registrati di mettere like all'evento
      • 2.1.4 Pagina di checkout
        • 2.1.4.1 Il Sistema Deve Mostrare agli utenti registrati i biglietti disponibili per l'evento e permettere la selezione
        • 2.1.4.2 Il Sistema Deve Permettere agli utenti registrati di completare l'acquisto dei biglietti selezionati
      • 2.1.5 Recensioni
        • 2.1.5.1 Il Sistema Deve Mostrare le recensioni relative all'organizzazione selezionata
        • 2.1.5.2 Il Sistema Deve Mostrare un resoconto del rating dell'organizzazione
        • 2.1.5.3 Il Sistema Deve permettere all'utente di lasciare al massimo una recensione per ogni evento dell'organizzazione a cui ha partecipato
      • 2.1.6 Profilo
        • 2.1.6.1 Il Sistema Deve Mostrare le informazioni del profilo utente
        • 2.1.6.2 Il Sistema Deve Permettere all'utente di modificare le informazioni del proprio profilo
        • 2.1.6.3 Il Sistema Deve permettere all'utente di modificare le impostazioni del proprio account
        • 2.1.6.4 Il Sistema Deve permettere ad un utente registrato di (smettere di) seguire l'utente di cui sta visualizzando il profilo
        • 2.1.6.5 Il Sistema Deve permettere ad un utente registrato di contattare l'utente se è un'organizzazione
        • 2.1.6.6 In tutti i profili il Sistema Deve Mostrare una sezione riguardo gli eventi a cui quell'utente ha messo like
        • 2.1.6.7 In tutti i profili il Sistema Deve Mostrare una sezione riguardo gli eventi a cui quell'utente ha partecipato
        • 2.1.6.8 Se il profilo è di un organizzazione il Sistema Deve Mostrare una sezione riguardo gli eventi creati da quest'ultima
        • 2.1.6.9 Il Sistema Deve Mostrare una sezione riguardo le bozze degli eventi nel proprio profilo di organizzazione
        • 2.1.6.10 Il Sistema Deve permettere ad un'organizzazione di vedere le statistiche di performance del proprio profilo se ha sottoscritto un piano di abbonamento
        • 2.1.6.11 Il Sistema Deve Permettere di visualizzare le recensioni (se presenti) nel caso in cui si stia visualizzando il profilo di un'organizzazione
      • 2.1.7 Biglietti
        • 2.1.7.1 Il Sistema Deve Mostrare i biglietti acquistati dall'utente potendo visualizzare il qr code e scaricarli in pdf
      • 2.1.8 Chat
        • 2.1.8.1 Il Sistema Deve Mostrare tutte le conversazioni tra utente e organizzazioni
        • 2.1.8.2 Il Sistema Deve Mostrare la chat con l'utente una volta selezionata la relativa conversazione, permettendo l'invio di messaggi
        • 2.1.8.3 Il sistema deve permettere di cercare gli utenti mostrando tra i risultati le conversazioni esistenti o permettendo di iniziarne di nuove
      • 2.1.9 Creazione Evento
        • 2.1.9.1 Il Sistema Deve Permettere di inserire tutte le informazioni sull'evento
        • 2.1.9.2 Il Sistema Deve Permettere di pubblicare/modificare/salvare come bozza/annullare/cancellare un evento
      • 2.1.10 Registrazione/Login
        • 2.1.10.1 Il Sistema deve permettere ad un utente di registrarsi o accedere alla piattaforma
      • 2.1.11 Scansione biglietti
        • 2.1.11.1 Il Sistema deve permettere di scansionare il qr code di un biglietto per validarlo, mostrando un errore in caso sia stato già utilizzato
      • 2.1.12 Sistema di navigazione
        • 2.1.12.1 Il Sistema di Navigazione deve permettere all'utente di navigare tra la schermata home, esplora, chat, profilo, biglietti e mostrare le notifiche. Nel caso di un'organizzaione deve essere possibile anche creare un nuovo evento e scansionare i biglietti
      • 2.1.13 Requisiti Non Funzionali
        • 2.1.13.1 Il frontend deve essere responsive
    • 2.2 Notifiche push

      • 2.3.1 Il sistema deve essere validato attraverso dei test di usabilità prima del rilascio sugli store

  • 3 Aggiornamento Sistema Esistente
    • 3.1 Registrazione con OAuth2
    • 3.2 Servizio Geolocalizzazione
      • 3.2.1 Il Sistema deve fornire un endpoint per ottenere luoghi data una stringa di query includendo il link a Google Maps
      • 3.2.2 Il Sistema deve fornire informazioni sulla posizione corrente una volta autorizzato
    • 3.3 Servizio Notifiche
      • 3.3.1 Il Sistema deve notificare un utente quando un insieme di utenti che segue ha messo like ad un certo evento
      • 3.3.2 Il Sistema deve notificare un utente quando un insieme di utenti che segue ha comprato i biglietti per un certo evento
      • 3.3.3 Il Sistema deve notificare un utente quando i biglietti per un evento a cui ha messo like stanno per terminare
      • 3.3.4 Il Sistema deve notificare un utente quando un evento a cui ha messo like si terrà a breve
      • 3.3.5 Il Sistema deve notificare un utente quando un evento per cui ha comprato i biglietti si terrà a breve
      • 3.3.6 Il Sistema deve notificare un utente quando i biglietti di un evento a cui ha messo like sono tornati disponibili
      • 3.3.7 Il Sistema deve notificare un utente abbonato quando il suo abbonamento sta per scadere se non ha attivato il rinnovo automatico
      • 3.3.8 Il Sistema deve notificare un utente se degli eventi vicino a lui si terranno a breve nel caso abbia attivato la localizzazione dall'app
    • 3.4 Piani di Abbonamento
    • 3.5 Servizio Ticketing
      • 3.5.1 Il Sistema deve permettere di creare codici sconto, da utilizzare in fase di checkout, agli amministratori
      • 3.5.2 Il Sistema deve permettere agli utenti di inserire un codice sconto in fase di checkout
      • 3.5.3 Il Sistema deve monitorare l'utilizzo dei codici sconto e renderne i dati consultabili dagli amministratori
    • 3.6 Frontend Web
    • 3.7 Requisiti Non Funzionali
      • 3.7.1 Ogni endpoint deve rispondere sotto i 200ms nel 99% dei casi
      • 3.7.2 Il frontend deve essere responsive
